# Contributing to CodeFortress

First off, thank you for considering contributing to CodeFortress! It's people like you that make CodeFortress such a great tool for the DevSecOps community.

## Development Setup

1. Fork the repo and clone your fork.
2. Ensure you have Docker and Python 3.9+ installed.
3. Install dependencies: `pip install -r requirements.txt` (once CLI is built).
4. Create a branch for your feature: `git checkout -b feature/amazing-feature`.

## Pull Request Process

1. Ensure any install or build dependencies are removed before the end of the layer when doing a build.
2. Update the README.md with details of changes to the interface, if applicable.
3. Your PR must pass all CI checks (Formatting, Linting, and Security Scans).
4. Obtain approval from at least one core maintainer before merging.

## Coding Standards
- Python: Follow PEP 8 guidelines. Use `black` for formatting.
- Documentation: Keep markdown files clean and update `docs/` for architecture changes.
